# tnotes ユーザーマニュアル

## 概要

tnotes はターミナル上で動作するメモ管理アプリケーションです。TUI モードと CLI モードの 2 つの使い方があります。

- **TUI モード**: Mac 標準の Notes アプリに近いレイアウトで、左にノート一覧、右にテキストエディタを表示します。フォルダ一覧を表示するとノート一覧の左にフォルダペインが追加されます。マウスとキーボードの両方で操作できます。
- **CLI モード**: サブコマンドでノートの一覧表示や内容の取得ができます。スクリプトやパイプラインとの連携に便利です。

## CLI モード

引数なしで起動すると TUI モードになります。サブコマンドを指定すると CLI モードで動作します。

```bash
tnotes              # TUIモードで起動（ソフトラップ有効）
tnotes --no-wrap    # TUIモードで起動（水平スクロールモード）
tnotes list           # ノート一覧を表示
tnotes list --trash   # ゴミ箱のノート一覧を表示
tnotes list --folder <name> # 指定フォルダのノート一覧を表示
tnotes purge          # ゴミ箱を空にする（確認あり）
tnotes purge --force  # ゴミ箱を空にする（確認なし）
tnotes get <id>       # 指定IDのノートを表示（ゴミ箱含む）
tnotes move <id> <folder> # ノートを指定フォルダに移動
tnotes create [file] # ファイルまたは標準入力からノートを作成
tnotes export <file> # データ一式をzipにエクスポート
tnotes import <file> # zipからデータをインポート
tnotes folder list         # フォルダ一覧を表示
tnotes folder create <name> # フォルダを作成
tnotes folder rename <old> <new> # フォルダをリネーム
tnotes folder delete <name> # フォルダを削除（確認あり）
tnotes folder delete <name> --force # フォルダを削除（確認なし）
tnotes help         # ヘルプを表示
```

### folder

フォルダの作成・削除を行います。

```
$ tnotes folder list
Notes                3
Work                 5
Personal             2
Trash                1
```

フォルダを作成します。

```
$ tnotes folder create Work
Created folder: Work
```

フォルダをリネームします。

```
$ tnotes folder rename Work Projects
Renamed folder: Work → Projects
```

フォルダを削除します。フォルダ内にノートがある場合は確認プロンプトが表示され、ノートはゴミ箱に移動されます。

```
$ tnotes folder delete Work
Folder "Work" contains 5 note(s). Notes will be moved to Trash.
Delete? (y/N) y
Deleted folder: Work (5 note(s) moved to Trash)
```

空フォルダの場合は確認なしで即座に削除されます。`--force` フラグで確認をスキップできます。

```
$ tnotes folder delete EmptyFolder
Deleted folder: EmptyFolder

$ tnotes folder delete Work --force
Deleted folder: Work (5 note(s) moved to Trash)
```

### list

ノートの ID、タイトル、更新日時を一覧表示します。

```
$ tnotes list
ID                TITLE          UPDATED
db7b7ce542656e1a  Hello, Notes!  2026-04-04 11:40
8a2816cfb412cd21  Meeting Notes  2026-04-04 11:33
```

`--folder <name>` フラグを指定すると、指定フォルダ内のノート一覧を表示します。

```
$ tnotes list --folder Work
ID                TITLE          UPDATED
db7b7ce542656e1a  Meeting Notes  2026-04-04 11:40
```

`--trash` フラグを指定すると、ゴミ箱内のノート一覧を表示します。

```
$ tnotes list --trash
ID                TITLE          UPDATED
f1a2b3c4d5e67890  Old Note       2026-04-03 09:15
```

### purge

ゴミ箱内のノートを完全削除します。実行前に確認プロンプトが表示されます。

```
$ tnotes purge
Permanently delete 3 note(s)? [y/N] y
Purged 3 note(s)
```

`--force` フラグを指定すると、確認なしで即座に削除します。

```
$ tnotes purge --force
Purged 3 note(s)
```

ゴミ箱が空の場合は何もせず終了します。

```
$ tnotes purge
Trash is empty
```

### get

指定した ID のノート本文を標準出力に出力します。`list` で確認した ID を指定します。通常ノートに加えて、ゴミ箱内のノートも取得できます。

```
$ tnotes get db7b7ce542656e1a
Hello, Notes!
This is a TUI app.
```

### move

指定した ID のノートを別のフォルダに移動します。移動先には「Notes」（デフォルトフォルダ）またはユーザー定義フォルダを指定できます。

```
$ tnotes move db7b7ce542656e1a Work
Moved to Work
```

Notes フォルダに戻す場合:

```
$ tnotes move db7b7ce542656e1a Notes
Moved to Notes
```

### create

ファイルまたは標準入力からノートを作成します。作成されたノートの ID が出力されます。

```
# ファイルから作成
$ tnotes create memo.txt
a1b2c3d4e5f67890

# 標準入力から作成（パイプ）
$ echo "Meeting memo" | tnotes create
b2c3d4e5f6789012

# ヒアドキュメントから作成
$ tnotes create <<EOF
Shopping List
- Milk
- Eggs
EOF
c3d4e5f67890abcd
```

1 行目がノートのタイトルとして扱われます。入力が空の場合はエラーになります。

### export

データディレクトリの内容（通常ノート、ゴミ箱含む）をzipファイルにエクスポートします。ディレクトリ構造がそのまま保持されます。

```
$ tnotes export backup.zip
Exported to backup.zip
```

出力先に既存ファイルがある場合はエラーになります。

### import

zipファイルからデータディレクトリにインポートします。既存データがある（`index.json` が存在する）場合はエラーになります。

```
# 新しい環境への復元
$ NOTES_DATA_DIR=/tmp/new-notes tnotes import backup.zip
Imported from backup.zip
```

export と import を組み合わせることで、データのバックアップや別環境への移行ができます。

## TUI モード

デフォルトではソフトラップ（行の折り返し）が有効です。長い行は画面幅で自動的に折り返して表示されます。`--no-wrap` フラグを指定すると、従来の水平スクロールモードで動作します。

### マウス

| 操作 | 動作 |
|------|------|
| ノート一覧ヘッダーの `≡` をクリック | フォルダ一覧を表示 |
| フォルダ一覧のフォルダをクリック | 該当フォルダを選択しノート一覧を切り替え |
| フォルダ一覧の `✕` をクリック | フォルダ一覧を閉じる |
| フォルダ一覧の `+` をクリック | フォルダ作成の入力モードを開始 |
| フォルダ一覧の `⋯` をクリック | フォルダ操作メニューを表示（ユーザー定義フォルダ選択時） |
| フォルダ一覧とノート一覧の境界をドラッグ | フォルダ一覧の幅を変更 |
| ノート一覧をクリック | 該当ノートを選択 |
| エディタヘッダーの `+` をクリック | 新しいノートを作成 |
| エディタヘッダーの `⋯` をクリック | エディタメニューを開閉 |
| エディタ領域をクリック | エディタへフォーカス移動、カーソル移動 |
| エディタ領域をドラッグ | テキストを範囲選択 |
| ノート一覧の境界をドラッグ | ノート一覧の幅を変更 |
| `Menu` ボタンクリック | フッターメニューを開閉 |
| `Copy` ボタンクリック | 選択テキストをクリップボードにコピー（選択時のみ表示） |
| `Cut` ボタンクリック | 選択テキストをクリップボードにコピーし削除（選択時のみ表示） |

### エディタヘッダーメニュー

エディタ上部のヘッダーにある `⋯` ボタンをクリックすると、ポップアップメニューが表示されます。

**通常モード:**

| メニュー項目 | 動作 |
|------|------|
| Delete Note | 選択中ノートをゴミ箱へ移動 |
| Pin Note / Unpin Note | ノートのピン留め ↔ 解除を切り替え |
| Move to… | ノートを別のフォルダに移動（フォルダ選択メニューが表示される） |
| Copy Note | ノート内容をクリップボードにコピー（本文があるときのみ表示） |

**ゴミ箱モード:**

| メニュー項目 | 動作 |
|------|------|
| Restore | 選択中ノートを復元 |

### フッターメニュー

フッター左端の `Menu` ボタンをクリックすると、ノート一覧上にポップアップメニューが表示されます。

| メニュー項目 | 動作 |
|------|------|
| Quit | アプリケーション終了 |

メニューを閉じるには、メニュー項目をクリックするか、メニュー外をクリックします。キー入力やウィンドウリサイズでもメニューは自動的に閉じます。

フッターのボタンおよびメニュー項目はマウスホバー時に背景色でハイライトされます。ボタンは罫線で囲まれた見た目で表示されます。テキスト選択中は `Copy` と `Cut` ボタンが `Menu` の横に追加表示されます。エディタで未保存の変更がある場合は `● Modified` が表示されます。

## ノート管理

### フォルダ一覧を使う

`Ctrl+B` またはノート一覧ヘッダーの `≡` ボタンでフォルダ一覧を表示できます。フォルダ一覧には「Notes」「Trash」およびユーザー定義フォルダが表示されます。

1. フォルダをクリックまたは `j`/`k` で選択すると、ノート一覧の表示がそのフォルダのノートに切り替わる
2. `Enter` / `Tab` でノート一覧へフォーカスを移動
3. フォルダ一覧を閉じるには `Ctrl+B` またはヘッダーの `✕` ボタンをクリック

### フォルダを作成・リネーム・削除する

1. フォルダ一覧のヘッダーにある `+` ボタンをクリック
2. フォルダ名を入力して Enter で確定（Esc でキャンセル）
3. フォルダをリネームするには、ユーザー定義フォルダを選択して `⋯` ボタンから「Rename」を選択し、新しい名前を入力して Enter で確定
4. フォルダを削除するには、ユーザー定義フォルダを選択して `⋯` ボタンから「Delete」を選択
5. フォルダ内にノートがある場合は確認ダイアログが表示され、承認するとノートがゴミ箱に移動される

「Notes」と「Trash」はシステムフォルダのため削除できません。

### ノートを作成する

1. `n` キーまたはエディタヘッダーの `+` ボタンでノートを作成
2. エディタに自動的にフォーカスが移動する
3. テキストを入力する（1 行目がタイトルになる）

### ノートを編集する

1. ノート一覧でノートを選択（`j`/`k` またはクリック）
2. `Tab`、`Enter`、またはエディタ領域クリックでエディタへ移動
3. テキストを編集する
4. `Esc` でノート一覧に戻ると変更が保存される

### テキストを選択・コピー・カット・ペーストする

1. エディタ内でマウスドラッグ、`Shift+矢印キー` で範囲選択、または `Ctrl+Shift+A` でテキスト全選択する
2. 選択範囲は青背景でハイライト表示される
3. `Ctrl+C` またはフッターの `Copy` ボタンで選択テキストをシステムクリップボードにコピー
4. `Ctrl+X` またはフッターの `Cut` ボタンで選択テキストをクリップボードにコピーしつつ削除
5. `Ctrl+V` でクリップボードからテキストをペースト（選択範囲がある場合は置換）
6. 選択中に文字を入力すると、選択範囲が入力テキストに置換される
7. `Esc`、別の場所をクリック、または矢印キー（Shift なし）で選択が解除される

### Kill と Yank

Emacs スタイルのテキスト操作が使えます。

- `Ctrl+K`: カーソルから行末までのテキストを削除して kill バッファに保存する。カーソルが行末にある場合は改行を削除する
- `Ctrl+Y`: kill バッファの内容をカーソル位置にペーストする

kill バッファはシステムクリップボードとは独立した内部バッファです。

### ノートをピン留めする

ピン留めしたノートはノート一覧の先頭に「Pinned」セクションとして固定表示されます。

1. エディタヘッダーの `⋯` メニューから「Pin Note」を選択
2. ピン留めを解除するには「Unpin Note」を選択

### ノートを削除する

1. ノート一覧でノートを選択
2. `d`、`Delete`、`Backspace` のいずれか、またはエディタヘッダーメニューの「Delete Note」でゴミ箱に移動
3. 確認なしで即座にゴミ箱に移動される

### ゴミ箱からノートを復元する

1. `g` キーでゴミ箱モードに切り替え
2. ノート一覧に削除済みノートが一覧表示される
3. エディタにノート内容が読み取り専用で表示される
4. `r` キーまたはエディタヘッダーメニューの「Restore」で選択中ノートを復元
5. 復元後は自動的に通常モードに戻り、復元したノートが選択される
6. `g` キーで通常モードに戻る

### 操作の取り消し・やり直し（Undo/Redo）

ノート操作（作成・削除・復元）とテキスト編集の両方で Undo/Redo が使えます。

- **ノート一覧**: `Ctrl+Z` でノート操作（作成・ゴミ箱移動・復元）を取り消し、`Ctrl+Shift+Z` でやり直し
- **エディタ**: `Ctrl+Z` でテキスト編集を取り消し、`Ctrl+Shift+Z` でやり直し

操作後にフッターに「Undo: Ctrl+Z」などのヒントが表示されます。

### ノートの並び順

ノート一覧はセクションヘッダー付きで日付ごとにグループ化されます。

- **Pinned**: ピン留めされたノート（ピン留めノートがある場合のみ表示）
- **Today**: 今日更新されたノート
- **Yesterday**: 昨日更新されたノート
- **Previous 7 Days**: 過去7日間に更新されたノート
- **Previous 30 Days**: 過去30日間に更新されたノート

各セクション内では更新日時の降順（新しいものが上）で並びます。

## トラブルシューティング

### 「Terminal too small」と表示される

ターミナルウィンドウを最低 80 カラム以上に広げてください。

### フッターのボタンが表示されない

ターミナルの高さが不足しています。ウィンドウを縦に広げてください。

### エディタでの編集内容が反映されない

`Esc` または `Tab` でノート一覧に戻ると、編集内容がノートに保存されます。アプリ終了時（`q`、`Ctrl+C`、`[Quit]`）にも自動保存されます。

## データの保存

### 保存場所

ノートは以下の優先順位で決定されるディレクトリに保存されます。

1. `$NOTES_DATA_DIR`（環境変数で直接指定）
2. `$XDG_DATA_HOME/tnotes/`（XDG Base Directory 仕様）
3. `~/.local/share/tnotes/`（デフォルト）

### ファイル構成

```
~/.local/share/tnotes/
├── index.json          # メタデータインデックス（起動時に使用）
├── Notes/              # デフォルトフォルダ
│   └── YYYYMMDD/       # 作成日ごとのディレクトリ
│       └── {id}.md     # フロントマター付きMarkdownファイル
├── Work/               # ユーザー定義フォルダ（例）
│   └── YYYYMMDD/
│       └── {id}.md
└── .trash/             # ゴミ箱ディレクトリ
    └── YYYYMMDD/
        └── {id}.md     # 削除されたノートファイル
```

各ノートファイルはフロントマター付きの Markdown 形式で保存されるため、外部のエディタやツールからも直接読み書きできます。

### 保存タイミング

以下のタイミングで自動的に保存されます。

- エディタからノート一覧へフォーカス移動時（`Esc`、`Tab`）
- 別のノートを選択した時
- アプリケーション終了時（`q`、`Ctrl+Q`、フッターメニューの `Quit`）

## 制限事項

- ソフトラップは文字単位の折り返しです（ワード単位の折り返しは未対応）

## ターミナルアプリの既知の問題

### Warp

- **URL クリック**: ソフトラップで折り返された長い URL をクリックすると、1 行目に表示されている部分だけが URL として認識されます。Warp は OSC 8 ハイパーリンクに対応しておらず、表示テキストからの自動 URL 検出に依存しているため、折り返し位置で URL が途切れます。iTerm2 / Kitty / WezTerm など OSC 8 対応ターミナルではこの問題は発生しません。なお `Ctrl+O` によるURL オープンはターミナルの OSC 8 対応に依存せず動作します
- **全角文字のハイライト**: エディタ内の全角文字（日本語など）にカーソルや選択ハイライトを適用すると、全角幅（2セル）ではなく半角幅（1セル）分しかハイライトされません。Warp のセル幅の扱いに起因する問題です
