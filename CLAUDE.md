## UIコンポーネントのファイル分割ルール

`internal/ui/` 配下のコンポーネントは以下の3ファイルに分割する。

| ファイル | 配置するもの |
|---|---|
| `xxx.go` | 構造体定義、コンストラクタ、getter、複数ファイルから呼ばれるヘルパー |
| `xxx_update.go` | Update 起点で呼ばれるメソッド（状態変更、スクロール、キー処理など） |
| `xxx_view.go` | View 起点で呼ばれるメソッド（描画、ハイライト適用など） |

### 配置の判断基準

- **呼び出し元で決める**: そのメソッドがどこから呼ばれるかで配置先を決める
  - 親コンポーネントの `_update` から呼ばれる → `_update`
  - 親コンポーネントの `_view` から呼ばれる → `_view`
  - 自コンポーネントの `_update` 内でしか使わない private メソッド → `_update`
- **複数の系統から呼ばれる場合は `xxx.go`**: `_update` と `_view` の両方、または初期化処理からも呼ばれるものは本体に置く
- **getter は `xxx.go`**: 状態を返すだけのメソッドは本体に置く
