# デモシナリオ

1本のデモで主要機能を紹介する。所要時間の目安: 60〜90秒。

## 前提

- `NOTES_DATA_DIR=/tmp/tnotes-demo`（空の状態で開始）

## シナリオ

### 1. 起動・議事メモの作成

- アプリ起動（空のノート一覧が表示される）
- `n` でノート作成、以下を入力:

```
週次ミーティング
議題: リリーススケジュール
結論: 来週月曜にv2.0リリース
```

- `Esc` でノート一覧に戻る

### 2. TODOノートの作成

- `n` で新しいノートを作成、以下を入力:

```
リリースTODO
リリースノート作成
テスト環境の確認
```

- `Esc` でノート一覧に戻る（Today セクションに2件表示される）

### 3. ピン留め

- リリースTODO を選択した状態で、エディタヘッダーの `⋯` → 「Pin Note」
- ノート一覧に「Pinned」セクションが表示される

### 4. フォルダ

- `Ctrl+B` でフォルダ一覧を表示
- `+` ボタンでフォルダ「Work」を作成
- Work フォルダを選択（ノート一覧が空になる）
- Notes フォルダに戻る
- `Ctrl+B` でフォルダ一覧を閉じる

### 5. 議事メモの編集（kill/yank）

- 週次ミーティングを選択、`Enter` でエディタへ
- 「議題:」行で `Ctrl+A` → `Ctrl+F` x4 で「議題: 」の後ろにカーソル移動
- `Ctrl+K` で「リリーススケジュール」を kill
- 「v2.0のスケジュール確認」を入力
- `Ctrl+N` で「結論:」行へ、`Ctrl+E` で行末
- ` (` を入力、`Ctrl+Y` で yank（「リリーススケジュール」がペーストされる）、`)` を入力

結果:
```
議題: v2.0のスケジュール確認
結論: 来週月曜にv2.0リリース (リリーススケジュール)
```

### 6. 全選択してコピー

- `Ctrl+Shift+A` で全選択
- `Ctrl+C` でクリップボードにコピー

### 7. ゴミ箱と復元

- `Esc` でノート一覧に戻る
- `d` でノートをゴミ箱に移動
- `Ctrl+Z` で取り消し（ノートが復元される）

### 8. 終了

- `q` で終了
